# Phase 2 — Demo Scenario Setup

Phase 2 seeds one repeatable, controlled Xero scenario so later phases have
real financial data to compare client requests against. No AI is involved —
this is just data.

## What the seed operation creates

Running the seed (`POST /api/demo/seed`, or the **Create demo scenario**
button on `/demo`) finds-or-creates, in this order:

1. **A default sales account** — the first active revenue account in your
   Xero organisation (preferring one with type `SALES`, e.g. the standard
   code `200`). Used as the `accountCode` on every seeded item and quote
   line. Not stored locally; re-resolved on every run.
2. **The demo customer**, Baker & Co.
3. **Five Xero Items** (the pricing catalogue).
4. **The original kitchen quote**, linked to Baker & Co.
5. **A local `DemoProject` record** linking `demo-kitchen` to the Xero IDs
   above.

Every seeded record carries a recognisable marker so it's easy to find in
Xero directly: the quote's reference is `SCOPELOCK-DEMO-KITCHEN-001`, and the
demo contact's email is `demo-baker@scopelock.local`.

## Original quote contents

Reference `SCOPELOCK-DEMO-KITCHEN-001`, title "Kitchen Refurbishment",
customer Baker & Co, total exactly £8,400 (no tax — every line uses Xero's
built-in `NONE` tax type so the total is deterministic regardless of your
organisation's default tax settings):

| Line | Qty | Unit price | Line total |
| --- | --- | --- | --- |
| Remove existing kitchen units | 1 | £600 | £600 |
| Supply and install new cabinets | 1 | £3,800 | £3,800 |
| Install oak worktop | 1 | £1,400 | £1,400 |
| Standard ceiling lighting | 1 | £350 | £350 |
| Install matte-black cabinet handles (`HANDLE-BLACK`) | 1 | £120 | £120 |
| Electrical work at existing socket locations | 1 | £2,130 | £2,130 |

**Total: £8,400.**

The wording is deliberately specific because Phase 3 will compare client
requests against this text semantically:

- "Install matte-black cabinet handles" is **already included** — so when the
  future demo message asks for matte-black handles, ScopeLock should
  recognise that as already agreed, not new work.
- "Electrical work at existing socket locations" explicitly does **not**
  cover *relocating* sockets — only work at their current locations. This is
  intentional: the future demo message asks to *move* two sockets, which is
  additional scope, priced from `SOCKET-MOVE` (£180 each × 2 = £360).
- LED lighting isn't mentioned in the original quote at all (only "Standard
  ceiling lighting" is), so a request for under-cabinet LEDs is additional
  scope, priced from `LED-PACK` (£240).

Expected Phase 3 result for the fixed demo message (see below): already
included — matte-black handles; likely additional scope — under-cabinet LED
lighting (£240) and relocating two sockets (£360); potential variation value
£600 (+ VAT if applicable).

## How duplicate prevention works

Every step searches before creating:

- The **contact** is looked up by `EmailAddress=="demo-baker@scopelock.local"`.
- Each **item** is looked up by `Code=="<code>"`.
- The **quote** is looked up by fetching Baker & Co's quotes and matching
  `reference === "SCOPELOCK-DEMO-KITCHEN-001"` client-side (Xero's Quotes
  endpoint doesn't expose a generic `where` filter the way Contacts/Items
  do).
- The **local project link** is `upsert`ed by its unique `slug`
  (`demo-kitchen`), never by customer name.

Re-running the seed any number of times reports `FOUND` for everything that
already exists and creates nothing new.

## How to run the seed process

1. Make sure Xero is connected (`/xero` shows "Connected").
2. Go to `/demo`.
3. Click **Create demo scenario**.

Or via the API directly: `POST /api/demo/seed` (no body needed).

## How to verify the data

Click **Verify demo scenario** on `/demo`, or `GET /api/demo/verify`. This is
read-only — it never creates anything, only reports what currently exists in
Xero and locally.

## Manual quote-status step

The seed tries to create the quote directly with status `ACCEPTED`. In
testing, Xero's Demo Company **silently created it as `DRAFT` instead**
(without erroring) — the code checks the actual returned status, not just
whether the API call succeeded, and flags this as `MANUAL_ACTION_REQUIRED`
when it happens:

> Open the quote in Xero (Business → Quotes, reference
> `SCOPELOCK-DEMO-KITCHEN-001`) and mark it **Accepted** to match the demo
> story ("an original accepted/agreed quote").

The quote is never sent automatically — only its status needs a manual
change in Xero's UI.

## How to reset local state

Tick the confirmation checkbox and click **Reset local state** on `/demo`
(or `POST /api/demo/reset-local` with `{"confirm": true}`). This deletes only
the local `DemoProject` row — **Xero contacts, items and quotes are never
deleted automatically**. Click **Create demo scenario** again afterward to
relink the same Xero records (they'll all report `FOUND`, not recreated).

## How to inspect the records in Xero

- **Contact**: Business → Contacts, search "Baker & Co" or
  `demo-baker@scopelock.local`.
- **Items**: Business → Products and services, search any of the 5 codes
  above.
- **Quote**: Business → Quotes, reference `SCOPELOCK-DEMO-KITCHEN-001`.
